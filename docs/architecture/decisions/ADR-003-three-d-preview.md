# ADR-003: 3D Room as an Optional Preview, Not the Whole Portfolio

## Status

Accepted by product direction

## Context

A full 3D portfolio would create avoidable performance, accessibility, SEO and usability risks.

## Decision

Use a dedicated low-poly 3D section inside the landing page. It displays an inline preview and opens to fullscreen after user interaction. All essential content remains available as normal HTML.

## Consequences

- memorable interaction without blocking recruiters;
- strict asset budgets required;
- mobile and reduced-motion fallbacks required;
- dashboard controls metadata and assets, not geometry editing.
