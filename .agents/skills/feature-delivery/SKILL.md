---
name: feature-delivery
description: Deliver a tracked portfolio-platform feature from feature file through implementation, tests, documentation, and final quality gates. Use for non-trivial FEATURE-ID work; do not use for tiny isolated edits.
---

1. Read root and nested AGENTS.md files.
2. Read the feature file and acceptance criteria.
3. Read relevant architecture, product and module documentation.
4. Ask the orchestrator to use explorer and architect subagents for non-trivial cross-module work.
5. Define file ownership before write agents start.
6. Use worktrees for concurrent independent writes.
7. Implement the smallest coherent solution.
8. Add or update tests.
9. Run targeted checks, then broader relevant quality gates.
10. Update documentation and feature status.
11. Return completed criteria, commands run, risks and unresolved work.
