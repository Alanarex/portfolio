---
name: repository-analysis
description: Analyze a GitHub or GitLab repository to detect technologies, architecture, tests, CI/CD, Docker, quality tools, activity, and safe portfolio metadata. Do not publish private source content.
---

1. Inspect repository metadata and visibility.
2. Read README and project documentation.
3. Inspect dependency manifests and lock files.
4. Inspect Docker, CI/CD, testing, static analysis and monitoring configuration.
5. Separate declared, detected and manually validated technologies.
6. Identify factual metrics only; do not invent impact.
7. Classify the project as featured, secondary, lab or excluded.
8. Mark demo and case-study suitability.
9. Protect private repository information.
10. Update research/content documentation with evidence and unresolved validation items.
